# ADR-P2-005: Tiered Semantic Answer Checking Strategy

**Status:** ✅ Accepted  
**Phase:** Phase 2 (System & Semantic Intelligence)

## Decision

Adopt a tiered answer-checking strategy that escalates from deterministic text matching to semantic similarity, using Sentence-BERT as the semantic layer.

## Options considered
1. **Exact or fuzzy matching only**  
   Rely exclusively on string-based comparison.
2. **Semantic-only checking**  
   Use a language model for all answer comparisons.
3. **Tiered strategy with semantic fallback**  
   Escalate from exact match → fuzzy match → semantic similarity.

## Selected option
Option 3 (Tiered strategy with semantic fallback)

## Rationale

The answer-checking mechanism is constrained by two primary requirements: maintaining a responsive gameplay experience and ensuring correctness for semantically equivalent answers.

Latency considerations drove the adoption of a tiered strategy. Most questions are factual types, where player answers differ only in formatting, spelling, or minor variations and can be resolved deterministically through exact or fuzzy matching. Escalating immediately to a semantic model would introduce unnecessary computation and latency on the critical gameplay path.

Correctness considerations determine the semantic fallback. When deterministic methods fail (for explanatory type questions), semantic similarity is required to recognize meaning-preserving rewordings. Sentence-BERT was selected for this role because it is explicitly trained for sentence-level similarity tasks and produces stable embeddings suitable for cosine similarity comparison. This provides robust semantic matching without introducing the complexity, cost, or unpredictability of more general-purpose language models.

Together, this design balances responsiveness and correctness by invoking semantic models only when necessary and by selecting the simplest model that satisfies the semantic requirement.


## Trade-offs

**Pros:**
- Fast and deterministic handling of common cases
- Semantic robustness for paraphrased answers
- Clear separation between cheap checks and ML-based checks
- Supports offline computation and low-latency runtime behavior

**Cons:**
- Requires threshold tuning for similarity scores
- Semantic similarity cannot resolve all ambiguous cases

## Risks

- Edge cases with very short or ambiguous answers may still be misclassified
- Similarity thresholds may require adjustment as the dataset evolves

## Consequences & Future considerations

- Establishes semantic similarity as a bounded, non-authoritative fallback
- Keeps runtime behavior predictable and explainable
- Enables a future SLM-based judge to be added as a final tie-breaker without replacing the existing strategy
