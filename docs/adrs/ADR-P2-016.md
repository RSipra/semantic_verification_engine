# ADR-P2-016: Manual Input Swimlane for High-Credibility Data

**Status:** ⏳ Deferred  
**Phase:** Phase 2 (System & Semantic Intelligence)

## Decision

Defer the implementation of a dedicated **manual input swimlane** for adding highly credible, low-volume questions to the dataset. The swimlane is intentionally designed but will be activated in a later phase once automated generation, validation, and quality control workflows are stable.

## Context

The Content Factory architecture includes a third input path (“Swimlane 3”) for manually authored questions. This path is intended for **high-credibility, low-volume inputs**, such as carefully curated baseline questions.

The manual swimlane serves a distinct purpose from automated generation:
- It provides a controlled way to introduce trusted, human-authored content  
- It supports the addition of one-off or creative questions that are difficult to generate reliably at scale  
- It enables occasional questions that create a different gameplay experience (e.g., cross-book or interpretive trivia)

This path is intentionally low-volume and exists to introduce novelty and variety rather than to materially rebalance the dataset or correct synthetic output at scale. Its value is qualitative rather than statistical.

At the current demo scale:
- Automated generation quality is still being characterized
- Adding a manual authoring path would increase implementation and operational complexity without immediate benefit

For these reasons, the swimlane is deferred rather than removed.

## Options considered

1. **Enable manual input swimlane immediately**  
   Allow direct manual addition of questions alongside synthetic generation.

2. **Exclude manual input entirely**  
   Rely solely on synthetic generation and automated correction mechanisms.

3. **Design now, defer activation**  
   Define the manual swimlane conceptually but postpone implementation until later phases.

## Selected option

Option 3 (Design now, defer activation)

## Rationale

The manual input swimlane is strategically valuable but not essential at demo scale. Early effort is better spent stabilizing the synthtic data generation and validation within the Content Factory. Deferring this swimlane avoids premature process overhead while preserving a clear upgrade path. When activated, it can be introduced without disrupting existing pipelines or quality guarantees.

## Consequences & future considerations

- The architecture explicitly reserves a path for high-credibility manual inputs  
- As synthetic generation volume stabilizes, the manual swimlane can be activated to:
  - introduce occasional, creative questions that enrich gameplay variety  
  - add carefully authored baseline or special-interest trivia  
  - provide a controlled outlet for human creativity within a finite content domain  

Deferring this swimlane is a sequencing decision, not a rejection of manual input as a long-term strategy.
