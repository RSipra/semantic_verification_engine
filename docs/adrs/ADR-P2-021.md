# ADR-P2-021: Deterministic Gold Master ID Assignment at Silver Tier

**Status:** Accepted  
**Phase:** Phase 2 (Semantic Verification Tracer)

## Decision
Assign `gold_master_id` at Silver promotion using a deterministic content hash derived from `question_type` + `question` + `answer`. The ID is immutable thereafter and carried forward to Gold via the publisher pipeline.

## Options considered
1. Sequential integer assigned at Gold promotion (original approach)
2. Deterministic content hash assigned at Silver promotion

## Selected option
Option 2 (Deterministic hash at Silver)

## Rationale
Two failure modes invalidated the sequential ID approach:
- **SBERT upgrade**: embedding recalculation regenerates Gold from Silver — sequential IDs cannot be guaranteed to reconstruct in the same order, breaking downstream references.
- **Gold/Silver corruption**: reconstruction from Bronze would produce different ID sequences with no recovery path without a separate ID registry.

Moving the ID to Silver and making it content-derived eliminates both failure modes. Gold becomes *idempotent*, i.e. it can be fully reconstructed from Silver at any time with identical IDs, with no auxiliary state required.

## Trade-offs
**Pros:**
- Gold is fully reconstructable from Silver (system invariant preserved)
- SBERT upgrades only require patching embedding columns -> IDs untouched
- Eliminates read-before-write dependency on existing Gold for ID assignment
- Single ID assignment point reduces pipeline complexity

**Cons:**
- Hash IDs are less human-readable than sequential integers
- Deduplication collision (two questions with identical type/question/answer) would
  produce the same ID — acceptable given Pydantic upstream deduplication guards

## Risks
- Hash collisions at scale: low risk for this dataset size; payload uniqueness is enforced by `source_quote` deduplication in the Silver validation pipeline.

## Consequences & Future considerations
- `gold_master_id` must be added to `cols_to_keep` in the publisher pipeline
- Silver Parquet schema and `SilverStandard` Pydantic model require a new `gold_master_id: str` field
- Gold concat is eliminated — publisher pipeline sheds columns and carries the hash ID forward only
- At full development scale, consider upgrading from MD5 (base4url) to SHA-256 if dataset grows beyond 100k questions
