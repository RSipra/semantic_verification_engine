# ADR-P2-011: Validation Layer (Pydantic)

**Status:** ✅ Accepted  
**Phase:** Phase 2 (System & Semantic Intelligence)

## Decision
Introduce a strict validation layer at the Silver dataset boundary using Pydantic schemas to reject malformed or invalid records before downstream enrichment.

## Options considered
1. **Implicit / ad-hoc validation**. Rely on downstream logic, heuristics, or runtime failures to surface malformed data. 
2. **Schema-based validation at pipeline boundaries (Pydantic)**. Enforce a single, explicit schema gate before data is promoted to the Silver layer.

## Selected option
Option 2 (schema-based validation using Pydantic)

## Rationale
The bronze-level generated questions can have errors even with careful prompting (e.g. format, structure, field consistency). A Pydantic validation layer as a gatekeeper to the Silver dataset ensures:

- Validation integrates naturally with Python-based pipelines
- Clear, actionable validation errors are surfaced early
- Only records conforming to an explicit schema are allowed to proceed
- Schema definitions serve as the single source of truth for a *valid* question record
- Downstream pipelines can rely on strong structural guarantees, protecting runtime stability and user experience

## Trade-offs
**Pros:**
- Early rejection of malformed or incomplete records
- Clear and consistent definition of valid data
- Simplifies downstream logic by reducing defensive checks
- Validation rules are centralized and explicit

**Cons:**
- Adds upfront strictness that may reject partially usable records. 
- Requires schema updates when data structures evolve

## Risks
- Validation errors may require manual inspection during early experimentation

## Consequences & Future considerations
- The Silver dataset becomes a reliable contract between generation and enrichment stages
- Downstream pipelines can assume structural correctness, improving stability and performance
- Schemas can evolve incrementally (e.g., optional fields introduced before being required)
- As the system stabilizes, validation rules can be hardened further to improve data quality guarantees
