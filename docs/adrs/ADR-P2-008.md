# ADR-P2-008: Structured Logging (JSON)

**Status:** ✅ Accepted  
**Phase:** Phase 2 (System & Semantic Intelligence)

## Decision

Use structured JSON logging for all runtime and pipeline components, integrating with Google Cloud Operations (Logging) rather than relying on unstructured text logs.

## Options considered
1. **Unstructured text logs**. Print-style or free-form log messages written to stdout.
2. **Structured JSON logs**. Emit logs as structured key–value records compatible with cloud logging systems.

## Selected option
Option 2 (Structured JSON logs)

## Rationale

As the system evolved beyond a local script into cloud-deployed services and automated pipelines, simple text logs became difficult to search, filter, and reason about—especially when debugging pipeline runs or runtime behavior across multiple executions.

Using structured JSON logs allows both the runtime application and the content factory pipelines to emit consistent, machine-readable log records, which is especially important for tracing failures and data flow across the Bronze, Silver, and Gold stages. This also integrates naturally with Google Cloud Operations (Logging), enabling query-based inspection by fields such as pipeline step, severity, run ID, dataset stage, or error type.

This decision was driven by the need for visibility and debuggability as the system scaled in complexity, rather than by performance or user-facing requirements.

## Trade-offs

**Pros:**
- Makes pipeline runs and runtime behavior easier to debug and audit
- Enables searchable, filterable logs in Google Cloud Logging
- Provides consistent logging across local development and cloud deployment
- Supports future monitoring and alerting without changing log format

**Cons:**
- Slightly more verbose log output than free-form text
- Requires discipline to maintain consistent log schemas

## Risks

- Poorly defined log fields could reduce the usefulness of structured logs
- Over-logging could introduce noise if not managed carefully

## Consequences & Future considerations

- Logging becomes a first-class system component rather than a debugging afterthought
- Pipeline execution, validation failures, and runtime errors can be inspected using a single tooling surface (Cloud Operations)
- As the system grows, structured logs can support metrics, dashboards, or alerting without redesign
- Log schemas may be refined as new pipeline stages or runtime behaviors are introduced
