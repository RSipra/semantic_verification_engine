# ADR-P1-001: Game design (SOC & MVC pattern)

**Status:** ✅ Accepted (Retroactive)<br>
**Phase:** Phase 1 (Discovery and Foundation)


## Decision
Implement strict **Model-View-Controller (MVC) separation**.

## Options considered
1. **Monolithic Script**: Single Python file mixing logic, data, and print statements. 
2. **Model-View-Controlloer (MVC) pattern**: Strict Separation of Concerns (SoC) between data, logic, and UI.

## Selected option
Option 2 (SOC with MVC pattern)

## Rationale

Separating the Game Logic (Controller), Data Handling (Model), and Input/Output (View) ensures that the core engine is environment agnostic. This allows us to port the game to a Web Interface in later design phases without refactoring the core game logic.

## Trade-offs
* **Pros:** significantly reduces technical debt during for the web interface migration.
* **Cons:** Requires slightly more boilerplate code initially compared to a simple script.
